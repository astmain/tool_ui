**Priority:** P1

**Files:**

- [ ] **Step 1: 补充公共类型导出**

在 `packages/components/src/types/` 下沉淀公共类型。

```ts
// packages/components/src/types/index.ts
export type U1Size = 'small' | 'default' | 'large'
export type U1Type = 'default' | 'primary' | 'success' | 'warning' | 'danger' | 'info'
export interface U1TableColumn {
  prop: string
  label: string
  width?: string | number
  fixed?: 'left' | 'right'
}
export interface U1MenuItem {
  key: string
  label: string
  disabled?: boolean
  children?: U1MenuItem[]
}
```

更新 `packages/components/src/index.ts` 导出这些类型。

- [ ] **Step 2: 拆分组件样式入口**

当前所有样式打包进 `style.css`, 按需加载时体积浪费。

在 `packages/components/src/styles/` 下为每个组件建独立入口:

```
styles/
  index.css        # 全量样式
  button.css       # 按需导入
  input.css
  dialog.css
  ...
```

更新 `packages/components/vite.config.ts` 输出多入口 CSS。

- [ ] **Step 3: 沉淀表单组件共享逻辑**

`U1Radio`, `U1Checkbox`, `U1Select` 存在相似逻辑:

- `disabled` 状态处理。
- `value` / `modelValue` 绑定模式。
- Group 子项通信机制。

创建 `packages/components/src/form/useFormControl.ts`:

```ts
export function useFormControl(props: {
  disabled?: boolean
  modelValue?: unknown
}) {
  const isDisabled = computed(() => props.disabled)
  return { isDisabled }
}
```

重构 Radio, Checkbox, Select 组件使用该 hook。

- [ ] **Step 4: 验证**

运行 `pnpm test` 和 `pnpm build` 确认重构未破坏现有功能。
